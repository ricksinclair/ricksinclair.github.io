Every AI-assisted workflow eventually hits the same wall: you've had a great session, made real decisions, figured out why something works the way it does — and then the context window resets. Next session, you start over. You re-explain the project, re-litigate decisions already made, re-derive context that cost you an hour to develop.

I tried long `CLAUDE.md` files. I tried elaborate Notion databases. They helped, but they didn't *reason*. Notion is a great capture surface and a terrible reasoning surface — you can't ask it "what across all my work connects to X?" and get a useful answer.

So I built Engram.

## What it is

Engram is a bi-temporal knowledge graph that lives in Neo4j, with Notion and local files as capture lanes. An AI agent ingests new material into the graph at the end of each session, maintains edges between concepts, and can answer questions across sessions because the graph persists where the context window doesn't.

The model is simple: every durable conclusion — a decision made, a rationale recorded, a pattern noticed — becomes a node. Relationships between nodes become edges. Time is a first-class property, so when something changes, you mark the old node `valid_until = now()` and create a new one rather than deleting history. The graph is always a faithful record of what was true *and when*.

## The key insight

**Memory and reasoning are different problems.** A vector store solves the first one (approximately). A graph database solves both — and you can read every query. There's no opaque blob, no mystery about why the system believes something. You can trace any answer back to the exact nodes and edges that produced it.

Most "AI memory" tools I looked at treated the whole problem as retrieval: embed your notes, do a similarity search, stuff the results into a prompt. That works until you need to answer relational questions — "what decisions did I make *because* of X?" or "which projects share this constraint?" — where similarity search breaks down and graph traversal shines.

## How it works in practice

At the end of a meaningful session, the agent writes Insight nodes capturing decisions and their rationale, merges Source nodes for any artifacts produced (Notion pages, repo docs, Claude memory files), and connects everything with typed edges. A `npm run resume` at the start of the next session runs a "where were we" query — freshness of the graph, newest insights, open threads — so the next session picks up exactly where the last one left off.

The whole stack is plain Neo4j + Node.js scripts + git. No proprietary framework to outgrow, no vendor to depend on. You can read every line.

The friendly getting-started walkthrough at [ulrictodman.com/engram/getting-started.html](https://ulrictodman.com/engram/getting-started.html) covers two tracks: one for people comfortable in a terminal, one for people who just use Notion and Claude.

If you've ever finished a great AI session and wished the next one could start from where you left off, Engram is for you.
